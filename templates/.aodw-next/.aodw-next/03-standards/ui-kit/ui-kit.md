# UI-Kit: 统一 UI 设计规范

> **设计目标**：看起来像"成熟产品"，而不是"AI Demo"。克制、冷静、耐看，更接近 Linear / Notion / Stripe，而不是 AI 工具官网。

---

## 1. UI 设计系统（Design System）

### 1.1 设计目标
- 看起来像"成熟产品"，而不是"AI Demo"
- 克制、冷静、耐看
- 更接近 Linear / Notion / Stripe，而不是 AI 工具官网

### 1.2 颜色系统（关键）

#### ❌ 严禁使用
- 蓝紫渐变
- 紫色主色
- 霓虹色、高饱和度红/蓝
- 科技感发光效果

#### ✅ 允许的主色方向（任选其一）
- 深灰 + 中性蓝
- 深灰 + 墨绿
- 深灰 + 暖棕
- 黑白灰 + 单一低饱和强调色

#### 推荐基础色
- Text Primary: #111827
- Text Secondary: #6B7280
- Border: #E5E7EB
- Background: #FFFFFF / #F9FAFB

### 1.3 字体与排版
- 字体风格：中性、无装饰
- 禁止大号粗体堆砌
- 所有页面必须有清晰层级

#### 层级示例
- H1: 20–24px / semibold
- H2: 16–18px / semibold
- Body: 14px / regular / line-height ≥ 1.6
- Meta: 12px / regular

### 1.4 布局与留白
- 使用 8px spacing system
- 页面必须有呼吸感
- 不允许"信息塞满屏幕"

### 1.5 组件风格
- Card：轻边框 / 无阴影或极弱阴影
- Button：扁平、低饱和
- Form：label + helper text 必须存在

### 1.6 整体风格关键词
> calm / neutral / restrained / professional / human-made

## 2. UI 生成提示词（Generation Prompt）

你现在不是 AI 工具展示界面设计师，而是在为一款真实上线的专业产品设计 UI。

### 2.1 必须遵守的规则

1. **禁止任何"AI 感"设计**
   - 不使用蓝紫渐变
   - 不使用发光、未来感、科技感
   - 不出现"AI 官网风格"

2. **设计目标**
   - 像 Linear / Notion / Stripe Dashboard
   - 克制、冷静、耐看
   - 看起来是人类设计师做的

3. **所有 UI 必须**：
   - 先给出结构与设计说明
   - 再给代码
   - 如果不确定好不好看，优先删减而不是增加

4. **UI 必须严格遵循本文件第 1 节（UI 设计系统）中的规范**
   - 不允许自行发挥审美

---

## 3. 禁止的 AI 风格（Forbidden AI Style）

以下设计一旦出现，视为失败：

- 蓝紫渐变背景
- 紫色 / 品红色主色
- 高饱和科技蓝
- 发光边框
- 未来 HUD 风格
- 赛博、量子、AI Lab 视觉语言

**出现以上任意一项，必须重做 UI。**

## 4. UI Critic Agent（UI 审查官）

### 4.1 角色定位

你是一个非常挑剔的资深 UI/UX 设计总监。

你的任务不是写代码，而是：
- 找出界面"像 AI 做的地方"
- 指出哪里丑、哪里廉价、哪里不成熟
- 强制要求简化和重构

### 4.2 评判标准

1. 是否有 AI 默认审美特征
2. 是否像真实产品，而不是 Demo
3. 是否存在颜色、排版、层级滥用
4. 是否"用力过猛"

### 4.3 输出要求

你的输出必须包含：
- **丑点清单**（明确指出）
- **为什么丑**
- **如何改**
- **是否建议整体重做**（是 / 否）

你可以非常严格，不需要安慰。

3. **人类设计感**：看起来是人类设计师做的，而不是 AI 生成的
4. **严格遵循规范**：不允许自行发挥审美，必须遵循本文件第 1 节的规范
